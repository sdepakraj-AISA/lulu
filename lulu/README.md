# Lulu

**Get discovered by AI agents. In 30 minutes. No code.**

Lulu is the universal AI discovery layer for SMBs. We auto-generate a live MCP server for every business and index them in the Lulu Registry — so when a customer asks ChatGPT, Claude, or Gemini for a recommendation, Lulu-listed businesses show up.

---

## Tech stack

- **Frontend:** Next.js 14 (App Router) + Tailwind CSS
- **Backend:** Node.js + TypeScript
- **MCP Runtime:** `@modelcontextprotocol/sdk`
- **Database:** PostgreSQL via Supabase
- **Hosting:** Vercel (frontend) + Railway (MCP runtime, coming Sprint 2)
- **Connectors:** Merge.dev unified API (Sprint 2)
- **Payments:** Stripe (Sprint 3)

## Local setup

### Prerequisites
- Node.js 18+ (`node --version`)
- A [Supabase](https://supabase.com) account (free tier is fine)
- Git

### Steps

1. **Clone the repo**
   ```bash
   git clone https://github.com/sdepakraj-AISA/lulu.git
   cd lulu
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Set up Supabase**
   - Create a new project at supabase.com
   - Go to SQL Editor → New query
   - Paste and run the contents of `supabase/schema.sql`
   - Copy your Project URL and anon key from Settings → API

4. **Configure environment**
   ```bash
   cp .env.example .env.local
   # Fill in your Supabase URL and keys
   ```

5. **Run locally**
   ```bash
   npm run dev
   # Visit http://localhost:3000
   ```

## Architecture

```
AI Agent
   │
   ├── GET /api/registry/search   ← Find a business (Lulu Registry)
   │
   └── POST /api/mcp/[businessId] ← Call tools on a specific business
          │
          ├── initialize
          ├── tools/list          ← Schema generated by schema-generator.ts
          └── tools/call          ← Executed by runtime.ts
                 │
                 └── Supabase (businesses, connectors, analytics)
```

**Key principle:** Agent-proof by construction. Prices, slots, and inventory are always sourced from the database. The agent cannot inject or override values.
